# 🧱 Nest.js Practice

Практичний розділ для вивчення **Nest.js** — прогресивного Node.js фреймворку для створення масштабованих бекенд-застосунків.  
Тут зібрані вправи, приклади архітектури, робота з модулями, контролерами, сервісами, базами даних і аутентифікацією.

---

## 📚 Структура тем

1. **Основи Nest.js**
   - Що таке Nest.js та його переваги
   - Установка CLI (`@nestjs/cli`)
   - Створення першого застосунку (`nest new project-name`)
   - Структура проєкту (Modules / Controllers / Providers)
   - Dependency Injection у Nest.js

2. **Контролери та маршрути**
   - Створення контролерів (`@Controller`, `@Get`, `@Post`, `@Param`, `@Body`)
   - Валідація запитів через `class-validator` та `class-transformer`
   - Використання Pipes і Guards

3. **Сервіси та провайдери**
   - Принцип інжекції залежностей
   - Створення власних сервісів (`@Injectable`)
   - Робота з інтерфейсами

4. **Модулі та структура проєкту**
   - Імпорт і експорт модулів
   - Поділ застосунку на фічі
   - Динамічні модулі

5. **Робота з базами даних**
   - Підключення до MongoDB (через `MongooseModule`)
   - Підключення до PostgreSQL (через `TypeORM`)
   - Створення ентіті, схем і репозиторіїв

6. **REST API**
   - CRUD-операції
   - DTO (Data Transfer Object)
   - Валідація та обробка помилок

7. **Аутентифікація та авторизація**
   - JWT (JSON Web Tokens)
   - `Passport.js` інтеграція
   - Guards (`AuthGuard`, `RolesGuard`)

8. **Файли, завантаження і зображення**
   - Використання `@nestjs/platform-express` та `multer`
   - Обмеження за типом і розміром файлу
   - Збереження у хмарі (Cloudinary, AWS S3)

9. **Вебсокети та реальний час**
   - Використання `@WebSocketGateway`
   - Підключення клієнтів і надсилання повідомлень
   - Чат або Live-оновлення приклади

10. **Тестування**
    - Юніт-тести з Jest
    - Тестування контролерів і сервісів
    - E2E-тести (`@nestjs/testing`)

11. **Розгортання**
    - Конфігурація для Render / Vercel / Railway
    - Експорт конфігурацій `.env`
    - Docker (Dockerfile + docker-compose.yml)

## 💡 Корисні команди

```bash
# Створити новий проєкт
nest new project-name

# Запустити сервер
npm run start:dev

# Створити контролер
nest g controller users

# Створити сервіс
nest g service users

# Створити модуль
nest g module users
